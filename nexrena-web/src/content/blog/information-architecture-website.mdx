Information architecture is the blueprint. Get it wrong and users get lost. Get it right and every page has a clear purpose and path.

## Sitemap First

- **Hierarchy** — Home → Sections → Pages. Logical flow.
- **Flat enough** — Don't bury important pages 4 clicks deep. 3 levels max for key pages.
- **No orphans** — Every page reachable from somewhere. No dead ends.

## Navigation

- **Primary** — 5–7 items. Work, Services, Industries, Resources, About, Contact.
- **Secondary** — Dropdowns or mega-menus for sections with many pages.
- **Footer** — Repeat key links. Services, industries, contact. Redundant is fine.

## URL Structure

- **Clean** — /services/web-design not /page?id=47.
- **Descriptive** — URL reflects content. /industries/manufacturing.
- **Consistent** — Same pattern across sections. No mixing.
