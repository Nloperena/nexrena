# Nexrena Design Principles — 8sleep + Apple Inspired

**Reference sites:** [8sleep.com](https://www.eightsleep.com), [apple.com](https://www.apple.com)

---

## Core Principles

### 1. Clarity & Deference (Apple)
- **Content first** — UI recedes; typography and imagery lead
- **Deference** — Design defers to content, not the other way around
- **One idea per section** — Clear, singular focus

### 2. Generous Whitespace (Apple)
- **Breathing room** — Sections need space to breathe
- **Padding** — Increase section-container padding; avoid cramped layouts
- **Negative space** — Use it intentionally, not as filler

### 3. Trust & Value (8sleep)
- **Trust badges** — Lead with clear value: "Free discovery call", "Weekly delivery", "100% retention"
- **Risk reversal** — Make the first step feel low-risk
- **Social proof** — Stats, testimonials, client names — but clean, not noisy

### 4. Minimalism (Apple)
- **Less is more** — Remove decorative elements that don't serve a purpose
- **Grain/texture** — Use sparingly or remove; Apple avoids texture noise
- **Borders** — Subtle; avoid heavy borders
- **Diagonal cuts** — Use sparingly; prefer clean section breaks

### 5. Typography Hierarchy (Apple)
- **Clear hierarchy** — Headline → subhead → body
- **Readable** — Body text 16–18px minimum, good line-height
- **Weights** — Use weight to create hierarchy, not size alone

### 6. Purposeful Motion (Both)
- **Subtle** — Animations should feel refined, not flashy
- **Fast** — 0.3–0.6s; avoid long, drawn-out transitions
- **Meaningful** — Every animation has a purpose

### 7. Section Structure (8sleep)
- **Defined blocks** — Each section has a clear role
- **Trust strip** — Optional horizontal trust badges near top
- **Feature → Benefit** — Lead with outcome, support with detail
